Overview

  • ViewCast is a part of the surface controller which is used in conjunction with Viewscape. This allows you to setup an RTSP stream of your desktop monitor (or an application running on the PC).

Functionality

  • ViewCast allows you to setup an RTSP stream when running the surface controller to stream a monitor or an application to the Webstreamer which then distributes the video across the network.
  • You can also setup the ViewCast RTSP stream within Viewscape and get the stream working within Surface Controller.

Pre-requisites

  • You will need Viewscape to add the RTSP stream to the camera list which allows you to stream.
  • You will need Viewscape Pro Server installed because this is what you use to setup ViewCast and to enable it within the options.
  • For testing purposes, you can use VLC to access the RTSP stream if you need to troubleshoot.
  • AN INTERNET CONNECTION IS REQUIRED TO ACTIVATE VIEWCAST, YOU ALSO NEED TO ACTIVATE VIEWCAST ON THE WINDOWS USER ACCOUNT IT WILL BE USED WITH (like Operator and not the Admin account).

Configuration

Configuring ViewCast

  • To configure ViewCast you will need to load up Viewscape Settings Editor, this is because you will need to add a license to the key as well as setup a username and password used.
  • Load up Viewscape Settings Editor
  • Go to the surface tab and then click surface001.

  • Then below that click surface001

  • Scroll to the bottom of the page and click ViewCast Key and add the key provided.

  • Here are the options provided within the Viewscape Editor which you can change, this guide is focussing on a basic configuration.
  • Add the key provided within the ViewCast Key category.

  • If the client wants to password protect their RTSP stream add a password to the Viewcast stream.

  • Add a username for the stream, such as Admin or another recognisable name then save the name of the website.

  • Save current page at the bottom of the display.
  • Start or restart the Viewscape Pro server to proceed.
  • Launch Viewscape and the surface controller.
  • After you have launched the surface controller as well as Viewscape go to the bottom right-hand corner on the taskbar and right click on the surface controller icon and click "Viewcast" to create a stream.

  • This is how to access the Viewcast options.
  • It will pop up with an authentication window where you're then required to insert the password you provided within the Viewscape settings editor.

  • Type in the password into the blank box.
  • You will then see a few sessions, which will most likely be displays. To access the stream settings, you will need to double click the sessions which appear in the windows.

  • The URL is for an RTSP stream, you can view this within VLC if you want to test or within a test window within Surface.
  • When you double click the sessions which appear (view the previous screenshot) you will get a settings menu where you can change some of the settings, such as the frame rate of the stream or if the password is required to view the stream.

  • To change some of the more advanced settings you will need to click the admin mode icon to change things such as the frame rate.
  • When you enable admin mode you will be prompted to enter the username and password you entered within the Viewscape settings editor.

  • You can now change the settings such as the frame rate (which I highly recommend, you don't want to hurt people's eyes) as one of the more basic options.

  • To find more settings you can click encoder settings.

  • The first options are JPEG and AVC (MPEG4 Part10 H264). This is type of video sent out by the encoder, JPEG will generally be a lower quality video compared to AVC.

  • Video usage allows you to change between 3 different options, the higher the quality more computing resources will be used.

  • Video profile allows you to change target quality level, high will look much better but it takes more computational power (run on baseline for now though, due to not knowing what devices it will be running on).

  • The target bit rate allows you to change the bit rate, the higher the bit rate the less quality is lost within the stream but this might cause network issues so this will be more site specific.

  • Key frame interval allows you to change the amount of "key" frames used or the main frames used to change the picture, the more key frames the better the image at the expense of more data being used.

  • The next options are variable bit rates and a constant bit rate. If you want to reduce the amount of traffic used over the network a variable bit rate will be preferred.

  • And finally, the Overlay settings box. This allows you to put text over the video and change the font of the text and where the text appears on the stream as well as the colour.

  • To test the stream to make sure that it's working correctly leave the encoder settings and click the test URL button near the RTSP address at the top of the page.

  • When you click test URL you will get a window which shows what is happening on the monitor or application you have assigned it to.

  • This is also what happens when the stream window appears on the monitor you're streaming.

Adding Camera Stream Authentication

You will need to setup camera stream authentication so you can log into the RTSP stream from Viewscape, if not the stream won't appear within surface.

  • Firstly, you will need to go to Viewscape and at the top you will need to go to admin.

  • You will then need to scroll down until you see lookups, which is at the top of the pile.

  • You will then see a window with a lot of options on the left-hand side, scroll down until you see "pro camera -- stream authentication". This is so you can log into the camera so you can access the camera feed.

  • Then you will see alias and the username and password, add the username and the password of the RTSP stream (this is the username and password you gave when you started up Viewcast).

  • Then click save and close.

Adding an RTSP Stream into Viewscape

Adding an RTSP stream to Viewscape can allow you to view the stream within Surface or on another machine. (It's basically like adding a normal camera to Viewscape).

  • To add an RTSP stream to Viewscape you will need to go to "information" then "camera".

  • Give the camera a name, such as Viewcast 1 or another identifiable name within Viewscape.

  • Then go to "Pro" within the window. And change the camera type to IP and more options will appear.

  • Under aspect ratio change the aspect ratio to "fill"

  • Under connection add the RTSP stream URL, as this is what is uses to contact the stream. (You can find this from the Viewcast window).

This is at the bottom of the Viewcast screen.

This is where you add the RTSP stream.

  • Add the camera stream configuration to Viewscape, this might be under the name of RTSP or Viewcast.

  • Add the camera stream authentication name, this is the username and password for the camera.

  • Then click save and close at the bottom of the window.

Debugging

  • If the program works for a limited amount of time, make sure that the key you put into Viewscape settings editor is correct as it will think it's a trial key which only lasts for a small amount of time.
  • In the alpha version if you click test URL and then close the window which appears to view the video also closes surface controller, VLC will work with RTSP streams for testing purposes.
  • If you have the Test URL window open on the same page as its streaming don't leave it running because it eventually crashes.
  • If you have configured the Viewcast as a stream within Viewscape and it doesn't appear in the camera list, make sure that you have the refreshed Pro Live.
  • If you have configured Viewcast as a stream, make sure that the stream is actually running.
