Overview

The purpose of this document is to provide detailed instructions for the integration of Stentofon into Viewscape.

Stentofon is a concierge system made up of servers, handsets and intercoms.

Requirements

Functionality

Viewscape is able to handle incoming and outgoing calls via the Stentofon handsets, providing the devices and required pro events are configured.

Pre-requisites

  • Viewscape Version 9.27.0 or greater

  • Pro Controls 2.8.1 or greater

Customer Requirements

This section contains a list of requirements which need to be obtained from the customer to configure the integration. In order to configure Stentofon into the Viewscape system, the following information is required;

  • Server IP Address + Port

  • Audio NVR IP Address, Username and Password

  • Pro events list for each intercom -- for incoming calls

Installation

The installation consists of

Viewscape Configuration

Adding a Stentofon Server and Importing Devices

Stentofon Server device is added in the following menu and then use then import tool to import configured Stentofon devices from Stentofon Server

Once all devices are added, Pro Server needs to be restarted.

Go to Admin -> VTAS Pro -> Device -> Device

Server and Audio NVR

![](screenshots/Stentofon Audio NVR/image2.png)Enter a name for the Stentofon Server and select the Device Type "Stentofon Server".

Enter the Server address in the format <ipaddress>:<port>

Default ports are 61112/61113.

Enter the IP address in Audio NVR in the Audio Address

Other ports may show as connected but may not allow calls to be connected

Port can be found out from Stentofon system.

Import Stentofon Devices

![](screenshots/Stentofon Audio NVR/image3.png)

Click on the Create devices button on Server form you have just added and wait approx. 20s for a message to pop up to notify what devices have been imported. Press this button again if new devices have been added to the Stentofon Server

Audio NVR

![](screenshots/Stentofon Audio NVR/image4.png) Go to Admin -> VTAS Pro -> NVR

![](screenshots/Stentofon Audio NVR/image5.png)

Click and select the newly added Stentofon NVR and edit to add the username and password supplied

Changing Handset Device Type and Assigning to Workstation (to be carried out only by Veracity Systems Personnel)

Viewscape imports all devices as Intercoms. Any handsets that have been imported require the device type to be changed from intercom to handset but this can only be done in SQL editing the Devices.tbl and the device type to 79.

Go to Admin -> VTAS Pro -> Device -> Device

![](screenshots/Stentofon Audio NVR/image5.png) ![](screenshots/Stentofon Audio NVR/image6.png)

Click and select a Stentofon Handset and edit and select the associated Workstation ID for Handset.

Setting up Pro Events

![](screenshots/Stentofon Audio NVR/image7.png) Create a pro event for every intercom and that needs to be answerable through Viewscape.

Pro events can be created for outgoing calls to allow for macro creation.

Pro Live

Add the Stentofon panel using the Pro Live Layouts tool.

Viewscape Pro Server

Stentofon will automatically be enabled in Pro Server if there are any server units configured in the database.

Temporary Fix -- If the site using stentofon is using call groups, enable StentofonCallGroup in Pro Server settings.

Note: This will render the system unable to work with non-callgroup stations.

Support

Troubleshooting

If any errors occur, try replacing the Stentofon.DLL in Pro Controls with the latest in the folder "Technical/Deployment/4 Integrations/Stentofon"

Failure when trying to connect call

![](screenshots/Stentofon Audio NVR/image8.png)

Verify pro server can ping the Stentofon server. If the Stentofon is on a different subnet, ensure the default gateway is entered into the routing section of the Stentofon server. Make sure the Port that is being used is correct
