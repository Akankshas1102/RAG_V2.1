Overview

  • Honeywell Morley is an alarm panel.
  • It's essential to note the debugging and pro server config specifies "Gent", however it is NOT a Gent panel, it is Morley.

Functionality

  • Receive alarms to trigger pro events.
  • Command the panel:
    • Reset
    • Silence
    • Accept
    • Evacuate
  • Currently only one panel can be connected to per pro server.

Pre-requisites

  • Rs232 connection module (EXP-005 RS232) must be installed in the panel and configured as 'Standard Morley Protocol' for communication with vtas. See panel installation manual.

Installation

Pro Server

  • Connect the panel to a serial port and alter VTASProServer.exe.config->GentSerialPort to match.
  • Pro server periodically polls the panel for new alarms. The default poll period is 5 seconds, but this can be tweaked by adding a: <number of seconds> to the GentSerialPort settings ie to poll every 10 seconds on comport 2 the setting would be 2:10.
  • No sdks are needed.

Viewscape Configuration

  • Add the alarm triggers to Admin->Vtas Pro->Alarm Panel->Device:

  • Each device on the panel has a loop/zone/address for identification.
  • Not all devices support all Gent Event Nature choices. 'Fire alarm signal' will be the most common alarm.
  • A single device can have more than one trigger ie a smoke detector may have a pre alarm and a fire alarm.

Troubleshooting

  • The panel comport is set to 9600 baud, 8 data bits, no-parity, 1 stop-bit by default and this is what pro server uses for the connection.

Debugging

  • Turn on pro server debugging VTASProServer.exe.config->DebugGent=True. Gent log lines are prefixed '(Gent)'.
