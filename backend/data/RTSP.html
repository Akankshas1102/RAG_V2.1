Overview

The purpose of this document is to provide detailed instructions for the integration of RTSP streams into Viewscape.

Functionality

RTSP Transport Support

ProtocolStatus
RTSP over TCPSupported
RTSP over UDPNot currently supported
RTSP over HTTPNot currently supported

Video Format Support

Only H.264 & H.265 compression is officially supported.

Other formats may be easily supported, but only those have been tested so far.

Pre-requisites

  • Viewscape Version 8.16 or greater
  • Microsoft Visual C++ Runtime (x86) 2015
  • Microsoft Visual C++ Runtime (x86) 2017

Customer Requirements

This section contains a list of requirements which need to be obtained from the customer to configure the integration.

Camera Stream Information

The required information for cameras in Viewscape is as follows;

  • Camera RTSP URL

This URL will vary between camera manufacturers. Consult the documentation for the target device to find it.

An example of what this will look like:

rtsp://10.128.14.220/h264main

Installation

Surface

  • No file copying is required, as used to be the case in previous releases. The integration files are fine to be left in Depends.

Note: As dependencies are all loaded from the 'Depends' folder, it is important that the Probing Path in StreamSurface.exe.config is correctly formed. An easy to miss error is a space before the path, just after the semi colon -- this causes failure in a very silent way. The LuxriotUserControl path MUST come before the Vemotion path because of some conflicts between the MediaFoundation.dll used in both SDKs.

Viewscape Configuration

Adding an RTSP camera

  • Create a stream configuration for RTSP cameras if none already exists. Set the camera type to 'RTSP'
  • Add a new camera ensuring that the stream configuration is correctly set.
  • Enter the RTSP URL for the camera into the "Stream IP address" field.

Troubleshooting

Common issues and steps to be taken are;

"component not initialised"

Ensure that the LuxriotUserControl dependencies are all copied into the root folder of StreamSurface, alongside the executable. Also make sure that the Visual C++ dependencies listed in the prerequisite section in this document are all correctly installed.

"video is just blank from all cameras"

Check the video drivers are up to date. Incorrect or extremely out of date drivers have been known to cause this issue.

"video is blank for just one camera"

Check the credentials for the camera stream are 100% correct. Incorrect credentials cause the stream to fail silently due to a lack of feedback from our RTSP integration. This will be addressed in a future update.

Debugging

To reveal more useful information as to why something may not work, the 'smart inspect' logs found in stream surface contain more detailed error information.

Alternatively running smart inspect whilst switching to an RTSP stream may provide real time error information that can help quickly identify the cause of the problem.
