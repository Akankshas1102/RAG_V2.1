Overview

Recorders

WCCTV Provide a public IP address for the cameras, and each camera will have its own unique port for Web HTTP video and NVR Port. These are both required for Viewscape to display live and recorded video

Functionality

Live and PTZ are achieved by directly requesting the images from the NVR (not the CCS). On the device itself both the WEB port and Web API need enabling in the options.

Playback and alarms in VTAS is done via the CCS. The Heitel CCS can be added to VTAS from within the lookups section, then specified in the NVR setup.

Export is not possible via the SDK.

Pre-requisites

  • Change Viewscape license key
  • Stream restrictions
  • The SDK contains all files required, these simply need to be installed into the correct place. This is done automatically in the installation of Pro Server and Surface. The SDK files should be found in the Depends folder.
  • Viewscape: requires the installation of the Heitel UC setup program.

Installation

  • Install the Heitel UC setup program to enable the playback features.
  • On the CCS locate the installation folder, make sure the following settings are set up in the file CAMCTRL.ini file:

VIDEOPORTON=1 (MUST BE 1 for video to pass through)

NUMOFCONN = n (number of video connections)

SMIMAGEMODE = 0 (to allow display of video)

HIDE = 1 (hides the video on the CCS software, improves performance)

** You can also determine the connection port to be used when setting up the CCS from within Viewscape (from within lookups) from here... It's by default port 5500. **

  • On each NVR ensure the WEB port is enabled and specified (default is port 80), and the Web API is also enabled.

Software

  • Install Heitel UC setup into "\i-Comply\Heitel UC"

Viewscape Configuration

  • Setup the CCS in Viewscape. Go to lookups->Heitel Servers. Add a new record, specify the IP and Port of the CCS (by default the port is 5500)
  • Add the NVR etc.
    • The port you should use on the NVR form is the web port. This is normally 80.
    • The devices serial number must be obtained. This can be taken from the NVR setup page. This is important as it's the only way we can establish which device an alarm has come in for.
  • For PTZ from the NVR the server setting 'HeitelHttpPTZ' must be set to true

Special Consideration for Ports

This can become slightly complicated because of the way the changes have evolved when it was realised that we needed another port for use with a WCCTV system using a 3G mobile network. In this case setting the StreamSurface.exe.config's CustomOptions or StreamSurfaceCustomOptions setting to 1 will alter the port used as the table below (hopefully) details:

Application/CaseDescriptionIP AddressPort
VTAS PlaybackVia the CCS. Using the CCS IP, CCS control port and the NVR IP.CCS IP, NVR IP (no port)CCS port setting
Pro Server (PTZ, Alarms)Via the CCS. Using the CCS IP, CCS control port and the NVR IP.CCS IP, NVR IP (no port)CCS port setting
Surface (Live) StandardTaken from the web interface.Only when specified in the NVR IP
CustomOptions = 1Taken from the web interface.NVR IPNVR PORT
StreamSurfaceCustomOptions = 1Taken from the web interface.NVR IPNVR PORT

My First Settings Editor

HeitelHttpPTZ -- Pro Server setting -- Determines if the PTZ is done via the web port or CCS

StreamSurfaceCustomOptions -- Surface setting -- Set to 1 to receive live video via IP Web Port

Support

Troubleshooting

  • Check network connectivity - web port is forwarded.
  • Check Heitel client works on CCS

Example NVR Configuration

In this example, we're using the Cam Control Server for Playback and PTZ. Live video is achieved from the Web port using HTTP requests.

  1. IP Address -- This is the public IP address and NVR port of the camera
  2. Port -- This is the Web port of the camera
  3. Heitel Server -- This is the camera control server configured in Admin -> Lookups
  4. Device Serial ID -- This is the private IP address of the camera obtained from the CCS

Ensure the NVR and Web ports use the same public IP address, otherwise only Live or Playback will work, depending on the IP address used.

StreamSurfaceCustomOptions -- Set to 1

HeitelHttpPTZ -- Set to False
