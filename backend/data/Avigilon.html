Overview

The purpose of this document is to provide detailed instructions for the integration of Avigilon products into Viewscape. Avigilon designs, develops, and manufactures video analytics, network video management software and hardware, surveillance cameras, and access control solutions. This primarily includes IP cameras and NVR's, and is integrated using the Avigilon Control Centre software (ACC).

All functionality that's part of the Avigilon integration is listed in the next section labelled "Functionality" and should be referred to before installation, along with the "Customer Requirements" section 1.3, which outlines all the required information to be obtained from the customer.

  4. Number of Recording Channels

Installation

Viewscape

For integration to Viewscape Workstations which require live, playback and exporting of clips from the Avigilon NVR, install the icomply Avigilon UC. This is done by copying the contents of the AvigilonUC.zip file in the root directory of this manual, into C:\\Program Files(x86)\\i-COMPLY\\AvigilonUC\\. Run the register.bat file to register the required dll's for the SDK. Run AvigilonControlCentreSDKRedistributable6.14.12.8.exe and install using defaults. This will install Direct x v9.0 which is also required.

Surface/Walls/FollowMe

If using Surface Controller, copy the contents of StreamSurface\\Depends\\Avigilon\\6.0.0.24 to the root folder of StreamSurface. This may not be included in the latest build due to file sizes and therefore is located in the deployment integrations folder as "Avigilon SDK". Run AvigilonControlCentreSDKRedistributable6.14.12.8.exe and install using defaults. This will install Direct x v9.0 which is also required.

VTAS

For Viewscape Pro Server, copy the contents of VTASProServer\\Depends\\Avigilon\\5.8.0.28 to root folder of ViewscapeProServer. If using motion detection events, set AvigilonAlarms to True in Settings Editor under ProServer tab. Run AvigilonControlCentreSDKRedistributable6.14.12.8.exe and install using defaults. This will install Direct x v9.0 which is also required.

Adding

The only requirement for adding cameras it the "Camera Type" on the camera record should be set to "Avigilon". This means the camera will be streamed directly off the Avigilon NVR for live video. Non-Avigilon cameras may be setup in the usual way in Viewscape, to provide the option of streaming from the camera or NVR for live video.

Adding

Avigilon recorders are configured through the NVR config page in Viewscape, and "NVR Type" set to Avigilon. When defining the channels, the cameras can be added in any order and the cameras are referenced by logical ID as specified in section 1.3.1, figure 1.

Troubleshooting

Common issues and steps to be taken are;

  1. Unable to display any live video from the NVR, but recording works fine

Avigilon implement two different streaming methods, labelled in their Control Center as "LAN" and "WAN". This isn't used in the conventional sense of the terms and WAN refers to the NVR being on a different subnet. LAN uses UDP and WAN uses TCP, although tests have shown in some cases LAN doesn't appear to work and therefore WAN should be used. If changing the connection method to WAN then you must be using VTAS 8.10+. The WAN option is selectable on the NVR config page.

Using

The Avigilon SDK comes with a test tool which is useful for verifying functionality and is named "DemoApp.exe", and also located in the Avigilon integration folder along with this manual.
