Overview

The Axis Virtual Camera software is used to create virtual cameras on the system, by using data from existing real cameras. It achieves this by using a recording of a real camera and looping the footage to be played back as live video. Each camera configuration is assigned the loopback address 127.0.0.1 along with a port number to access the camera.

Pre-requisites

There are no pre-requisites, however pre-configured setup is located on the NAS; \\\\172.16.10.5\\Media\\AXIS Virtual Camera.

Axis Configuration

Recording the footage from live -- Select the record button and input the IP Address and credentials of the Camera that you wish to record from. The Credentials will usually be: U: root P: m00se

Leave the Channel and format as they are unless you have a specific reason to change them.

Leave the Frame rate and resolution as they are too

Choose where to save the file and add the name to the end. Make sure you have permissions to save in the location you choose or the file will not be created. The file name can be anything that relates to the recording and ad a description if you wish

Clicking finish will start the recording

Click stop when you want to stop recording

Streaming the Recording - First you need to add a camera as a template to put recordings on. Click add and choose real camera.

Input the information about the camera: the IP Address and the credentials Press next

Press next again

Click finish to add the camera.

Now you need to double click on the camera you just added and go to Playback in the menu that pops up

Click on the live streams at the bottom and remove them

Then click on the Add button under the recordings section and choose the file that you wish to play.

Click OK

To start the playback on the camera just click on it once and click start on the top menu. To check the camera is working type in a loopback address in your browser followed by the port that is visible under the IP Addresses bar in AVC e.g 127.0.0.1:40001

VTAS Configuration

Open VTAS and add a new camera, Give it a name and a Pro ID. The IP Address should either be the IP Address that AXIS Virtual Camera is running on followed by the port shown on AVC e.g. 192.168.249.100:40001. The stream configuration should usually be set to whatever type of camera that the template is based on including the correct credentials

Support

There is no particular maintenance needed but it is important to make sure that the program stays open in the background until you are finished using it or the streams will stop

Troubleshooting

If the camera does not function in VTAS then there are a few steps to take to see what the issue could be;

  • Make sure that the Virtual Camera has started streaming
  • Check that the IP address is correct and that you can reach the Virtual Camera through a Browser
  • Check that the credentials are set up correctly in VTAS
