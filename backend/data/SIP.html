Viewscape 11.37.1 Manual

SIP

Some notes on configuring the Viewscape SIP integration.

Prerequisites

  • Viewscape v10.10.1 or higher
  • Pro Controls v3.2.6 or higher
  • Ozeki SDK License Key

Veracity Systems have an Ozeki License that can be used, details of which are in the SIP integration folder on the technical dropbox, but if another license is needed the key needs to be generated using Ozeki's license generator, which is installed along with the development sdk - also in the dropbox folder, or can be downloaded from here.

SIP Configuration

Viewscape will need to be restarted after configuring any of these steps to take effect.

The Ozeki License details need to be set in the Pro Controls settings, under OzekiUserID and OzekiLicenseKey.

SIP Device

To provide the connection details for the SIP Phone control, a device of type Viewscape SIP Phone.

)

  • Workstation
    • Linking this device to a workstation will mean that when a user logs into a workstation and does not have a custom SIP device configured for their User (see custom configs), this device will be used to provide the connection details for the SIP Phone Control.
  • SIP Exchange Address
    • The address of the PBX/VOIP server, can be IP or domain name.
    • If using a domain name only enter the domain and any subdomains, no protocols such as http:// or sip://
      • ✅ sip.voipfone.net
      • ❌ http://sip.voipfone.net/
  • SIP Port
    • The port used to connect to the SIP Exchange.
    • If left blank the default SIP port - 5060 - will be used.
    • NOTE - This is the port that will be used for SIP protocol messages only, and calls will use a port range defined by the SIP exchange, this cannot be configured via Viewscape.
  • SIP Username
    • This is the login used to register with the server, and can vary in format between different providers.
    • With the VOIP provider for the company, voipfone, this is in the format XXXXXXXX*YYY, with the YYY being the extension for that specific login and the XXXXXXXX being the overall login code for the company account. The entire XXXXXXXX*YYY is required in this field.
  • SIP Display Name
    • This is the name that will usually be displayed as the caller ID from this login.
    • NOTE - This is sometimes required to be exactly the same as the SIP Username field.
  • SIP Password
    • The password used to register with the SIP Exchange.

User Custom SIP Devices

Viewscape SIP Devices can be configured on a per user basis, so when logging into any workstation, the user's custom SIP Config will be used. This will override any workstation-specific SIP config, and use only the custom user config.

To do this configure as detailed above, but leave the workstation link empty, unless you intend for this device configuration to be the default on a specific workstation as well as a custom user configuration.

Once the device is created, it can be linked in the User config under the 'Pro' tab, in the field Viewscape SIP Phone, shown below.

Phone Control

To add the Viewscape SIP Phone to Pro Controls, add a panel of type SIP Phone to a layout.

Address Book Entries

The Phone Control includes a built in addressbook, which is populated from three different types of records in Viewscape: - Viewscape SIP Phone Device - Any configured Viewscape SIP Phone devices will be shown, using the Display Name configured, and the username as the number. - Generic SIP Device - A device type of Generic SIP Device can be configured with a Display Name and a Username(phone number). - These details are only used to populate the Address Book. - This device also contains event states that will be updated only if called by, or calling a Viewscape SIP Phone. External calls to and from these devices are not reported to us via the SIP Exchange and so their states cannot be determined. - As of version 10.10.1 this is erroneously listed as Viewscape PBX Server - will be corrected in the future, with no reconfiguration needed. - Viewscape Person Record - The telephone field on the person record is used to populate the SIP Phone Address Book, if this field is empty the entry will not be displayed.

Groups

Groups are defined within the SIP Phone Control. To edit them you must be logged in as a user with the "Edit System Settings" permission. If you have the correct permissions, a settings button will appear above the groups list in the Address Book Tab.

Pressing this button will open the Edit Groups dialog.

The sections show the addressbook (A) and the currently defined groups (B+C). Selecting a group in the Groups list (C) will show the members in the Group Members section (B).

  1. Delete a member from the selected group. This will remove the selected member in the Group Members List (B) from its group selected in the Groups List (C).
  2. Add a member to the selected group. This will add the selected member in the Address Book (A) to the group selected in the Group List (C).
  3. Delete Group. This will delete the group selected in the Groups List (C), and all members of that group.
  4. Add New Group. This will pop up a dialog for the user to enter a name for the new group, this will then be added to the Groups List (C).
  5. Save. This will apply all changes made in the dialog to the database.
  6. Cancel. This will discard any changes made in the dialog, which will then be unrecoverable.

Usage

Once all configured, the SIP Phone control will automatically attempt to connect using the details provided when opened. The control will display details of this, and be very apparent on the display if the connection has failed, as well as updating the device state. To force the control to attempt to reconnect, press the "call" button while the display shows its state as "Disconnected".

Address Book

Clicking on an entry in the Address Book will input the number to the SIP Phone Display, and can then be called as normal.

Additionally, if a number manually typed into the field or from an incoming call matches a record in the addressbook, this will be shown on the display.

Audio Device Config

The Audio Device Config tab includes two dropdown menus to choose which audio devices the phone will use for calls. These are initially set from the default audio device and microphone as defined within Windows. Changing these will immediately swap the phone to use the new device, including during calls. When Viewscape is restarted this will return to the Windows default.

Audio File Playback

The audio file playback tab allows the user to play an audio file into the call they are currently on. The file once finished will loop automatically and requires stopping manually if required. While playing, the microphone will still transmit as normal.

    1. Play/Pause the currently selected file.
    1. Stop the current playing file, will unload the file.
    1. Add a new file to the list, will open a file browser.
    1. Currently playing file display, will show the file loaded/playing and current progress.
    1. List of files, selecting one and pressing play will start playing the file into the call.

The list of files is saved locally automatically in a file named sipFiles.json, located in the same folder as the Viewscape.Sip.Phone.dll file, most likely in the root of VTAS Controls.

There is currently no synchronisation of this file between workstations, although once created this file can be copied to the same location on another workstation and will be automatically loaded as normal.

Group Calls

Note: The sdk that we have used for this integration has a license limitation on how many concurrent calls can be made.

There are two ways to start a group call from the SIP Phone Control.

  1. Double clicking a group in the group list will start a call with all of its members.
  2. Pressing the add to call button while already in a call will add the currently typed number to the call, converting it into a group call. Pressing this while not in a call will do nothing.

While in a group call, usual functions work the same as usual including playing audio via audio files, and ending the call with the "End" button. However there is a little bit of extra functionality to give you more control. When in a group call, instead of showing who you are in a call with in the top left corner, a list of participants will appear in the screen section of the phone. Hovering over a member of the call will reveal an 'X' button, pressing this will eject that person from the call. The list will also show the status of everyone in the call, it is colour coded but you can hover over for more info.
