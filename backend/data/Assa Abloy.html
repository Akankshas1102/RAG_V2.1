Overview

The purpose of this document is to provide instructions for the integration of Assa Abloy systems into Viewscape.

Functionality

Supported Functionality

  • Monitoring of Controller and Door status in Viewscape
  • Triggering alerts via Pro Events based on Assa Abloy system events
  • Basic door functions:
    • Pulse Open
    • Force door open (on/off)
    • Force door close (on/off)

Pre-requisites

  • Viewscape Version 8.30 or above, Viewscape Controls 1.8.4 or above.
  • Integration is via a protocol, there is no SDK to install.
  • The only dependency is AssaAbloy.dll that can be found in the Depends folder and should be referenced correctly by Pro Server already by the usual probing path method in the config.

Customer Requirements

This section contains a list of requirements which need to be obtained from the customer to configure the integration.

Assa Abloy Server Details

The customer or installer must be able to provide the following information on the Assa Abloy Server that Viewscape Pro Server will communicate with over https.

  • IP4 Address
  • SSH Event Port (default is 5003)
  • Credentials (default is admin account is master:master)

Installation

Viewscape Pro Server

Configuration

Configure the following settings on Viewscape Pro Server to allow communication with the Assa Abloy Server:

  • AssaAbloyServerIP
  • AssaAbloyServerEventPort
  • AssaAbloyUsername
  • AssaAbloyPassword

In Addition:

AssaAbloyStatusRefreshSecs

The time in seconds between each refresh of the door statuses. The default is 10 secs (safe). This should be reduced normally to around 5 secs, but it will depend on the available performance of the Server and number of Doors on the system.

The refresh will trigger first communication with Assa Abloy, these results are then parsed and the DB table AssaAbloyDoor_TBL is then updated and all Viewscape Clients are informed of this update. Viewscape Clients will then refresh their data from this table.

So, finding the right balance between update frequency and performance is crucial.

DebugAssaAbloy

Turns on more verbose logging within the log. All messages will have the following prefix:

[Assa Abloy]:

Operation

  • On start up of the Pro Server, a connection will be made using the details from the settings. This will initialise the maintenance of the AssaAbloyDoor_TBL table in the DB and create a constant connection to the Assa Abloy event stream over TCP/IP.
  • If the connection is lost for any reason (network fault, Assa Abloy server reboot etc.) Pro Server will continue to try and re-establish the connection every 5 seconds.

Alarms and Pro Event Configuration

Introduction

  • Be warned setup of Pro Events can be more complicated than usual due to the nature of the events provided by the Assa Abloy system.
  • The supported events, IE. Assa Abloy events we currently support are automatically loaded into the table AssaAbloyEvent_TBL. This is to increase overall performance of Pro Server by not checking events we aren't interested in such as a new user or a change in the configuration of the system, and also to provide the user with a list of events they can choose from when setting up the Pro Events. -- The list is vast.
  • If the client requires an additional event that is not in the list, then it can be added to the database by an engineer without needing a software upgrade. The current list was provided from the Assa Abloy documentation with a few obvious errors fixed.
  • Pro Events can optionally be setup for specific objects (EG. Doors/Controllers/users etc.). This requires a little and potentially access to the Assa Abloy client as I will discuss more of in the section below.

Pro Event Setup Basic!

  • First select "Assa Abloy" from the trigger field on the Pro Event form, then select the Assa Abloy event you would like to trigger from the list.
  • Pro Server will raise an alert from the Pro Event when it detects any instance of the Assa Abloy event specified.

Pro Event Setup Advanced

If you would like to setup more advanced events, then you can optionally use the type and value fields. NOTE: You must either leave both blank or specify them both.

If you aren't familiar with the Assa Abloy client and the event type-value arguments that are included on events then you will have to access the Assa Abloy client and view the logs to see what type and value arguments you might require.

Troubleshooting

Common Issues

  • Pro Events not triggering: Verify the event in the Assa Abloy client as demonstrated above. If you have specified an argument, check the case. These settings are case-sensitive.
  • Required Assa Abloy event available in the list: You will need to manually insert the event into the database table AssaAbloyEvent_TBL. You may need to ask someone for help if you aren't competent or comfortable doing this. Every effort has been made to include by default all likely events and more.
