Overview

The purpose of this document is to provide step by step instructions on how to configure motion detection alerts to be sent to VTAS from an Axis IP camera or encoder. Although it has been created primarily for engineers, it is possible for non-technical users to follow these instructions, however the Axis listening Port is required to be configured by i-comply trained engineers before the alerts will work.

The Axis listening Port is a settings node contained within the VTASProServer.exe.config file, and a recommended value to use is 501.

Configuration

The configuration is split into the following stages;

  1. Axis camera configuration (Webpage)
  2. VTAS configuration (Pro Events)

Axis Configuration

The Axis configuration is then carried out in the following order;

  • Configure Motion Detection (Detectors -> Motion Detection)
  • Configure the recipient of the intended alert (Events -> Recipients)
  • Configure the trigger and alert to be sent (Events -> Action Rules)

Configuring Motion Detection

Enter the Axis IP camera's / encoder's IP address into the address bar of internet explorer, and from there select Setup at the top of the page. From the main setup page, select Detectors from the navigation pane on the left. Once the detectors menu is expanded, select Motion Detection, as per figure 1 below:

Figure 1: Motion Detection menu

From this page you can add multiple motion detection windows and each window would need to have its own alert and trigger (pro event in VTAS). This can also be customised to exclude a window and everything else will be used for detecting motion.

The three settings on the right hand side allow for the adjustment of motion detection, being the size of the objects detected, history and sensitivity. The activity window below that indicates whether it is currently detecting motion. When the graph is blue, the motion is not great enough to be detected for the alert, and when it is, becomes red and is above the defined line.

Configure the Recipient

Without a recipient configured, the Axis IP Camera / Encoder has nowhere to send the alert to. The configuration is carried out in the Events menu on the left of the main setup page, and then when expanded, select Recipients, and then Add if the required recipient isn't configured, as per figure 2 below;

Figure 2: Recipients menu from Events

From the add button, the recipient setup is then displayed and requires a name, which for uniformity, should be the same as the hostname of the recipient machine. Set the type to TCP and enter the network address of the recipient, followed by the port number configured in the VTASProServer.exe.config file as mentioned at the beginning of this manual.

Figure 3: Recipient setup

Configure the Trigger and Alert

The next step is to configure the trigger and alert itself, which is to be sent to the recipient. This is achieved from the Events menu on the left of the main setup page, select Action Rules and then click add, as per figure 3 below;

Figure 4: Action rules menu from Events

The action rules setup menu will then appear. In the General section, make sure the rule is set to enable and enter a name for the rule, this has no relevance to any further configuration.

Under the Condition section, select Detectors from the Trigger drop down menu, and then select Motion Detection. If multiple motion windows have been configured, choose the relevant window from the drop down menu. If the alert is to be triggered whenever there is motion, select Always in the drop down menu for Schedule, otherwise select a configured schedule.

Figure 5: Action Rule Setup, for general and condition settings

In the Actions section, select Send Notification from the Type drop down menu and select the recipient as the same entry and configured previously in figures 2 and 3.

The message to be sent can be anything of relevance, providing it matches the entry in the Event ID field in the VTAS Pro Event configuration. The message MUST however, start with axe, and end with the @ symbol, for example axe,<enter message>@, as per figure 6 below (axe,MotionCam101@).

If the alert is be triggered constantly while there's motion, in the use of where video is displayed constantly while motion is present, then the notification needs to be sent before it times out. If using an Alarm Expander, then this setting is located in the VTASProServer.exe.config under AlarmExpanderDefaultTimeout. Typically though, set the notifications to send every 5-10 seconds.

Figure 6: Action Rule Setup, for actions settings

VTAS Configuration

  • How to configure it in VTAS

From within VTAS, select Admin from the top and then VTAS Pro, followed by Pro Event. The Pro Event configuration page should display.

Enter an Alias for the pro event and from the Pro Alarm Type drop down menu, select a category for the alarm.

Trigger

Set the Trigger as Axis from the drop down menu and then enter the Event ID which is the same as configured in the message of the Axis notification, for example MotionCam101 as entered in the example.

Figure 7: Pro Event trigger section

Alert

From the alert tab within the pro event configuration, enter a message to be displayed for the alert, which can be anything and is displayed in the alert stack. Select an incident category from the drop down menu for logging purposes and choose whether the alert should be; Reactive, Non Reactive, or Not Logged. It is recommended the alerts for motion are not logged however, as there tends to be a large volume of alerts generated from motion detection.

If an alarm expander is being configured for use on a Wonderwall controller, then enter the ID of the Wonderwall controller and select the camera which the motion is configured on. If a Macro is to be run instead, then select this from the Macros section in the bottom right corner of the page.

Figure 8: Pro event alert section
