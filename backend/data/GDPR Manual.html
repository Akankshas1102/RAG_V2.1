Viewscape 11.37.1 Manual

GDPR Manual

Introduction

The General Data Protection Regulation (GDPR) is a regulation in EU law that aims primarily to give control to citizens and residents over their personal data. Personal data must be stored using full anonymization, and use the highest-possible privacy settings by default, so that the data is not available publicly without explicit, informed consent, and cannot be used to identify a subject without additional information stored separately.

GDPR in Viewscape

Viewscape caters for GDPR by providing specific functionality for warnings and anonymization of personal data. This includes:

  • Warning that a person has been in the database over a (configurable) period of time.
  • Mute warning for a predefined extension period ie 1 week.
  • Anonymize a person. This should remove all information about the person, but will leave an anonymous/blank record in the database to maintain database integrity.
  • View all incidents a person is involved in.
  • Anonymize an incident. This should remove all information about the incident, but will leave an anonymous/blank record in the database to maintain database integrity.

Viewscape Usage

Person creation

When a person is created in Viewscape, this date is recorded with the person. This cannot be edited:

Data Retention Warning

Due to restrictions in the design of Viewscape, the warning period configuration is set directly within database. Set [SystemSettings_TBL]->[sysGDPRWarningPeriod_LNG] to the number of days to warn that a person has been held within the Viewscape database. The default is currently 10 years as this feature has not been signed off by customers yet (version 8.16).

On the Organiser there is a new section labelled 'Person Past GDPR Warning Period' in the Critical section. This includes all people that require attention. Double clicking the person or pressing the Detail button will open the Person form to perform an appropriate action to comply with GDPR.

Muting GDPR warning

On the Person form is a new tab labelled GDPR. To mute the warning to appear at a later date, choose a reason and an extension period, then press the Extend Warning button. The reason lookup is user configurable using Admin Lookups. The period is predefined as either 1 day, 1 week, 31 days or 365 days.

The extensions will be logged and further extensions are allowed if needed.

Person Incident Involvements

On the Person form go to the Involvements tab to see all the incidents a person is involved in.

Data Anonymization

Both the Person and Incident Forms have a Delete button. This will anonymize all the personal information, but keep a blank record within the database to maintain referential integrity.
