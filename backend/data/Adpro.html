Overview

The purpose of this document is to provide instructions for the integration of AdPro FastTrace systems into Viewscape.

Functionality

Supported Functionality

  • Live Video and Audio
  • Replay Video
  • Alarms on ProServer (via AdPro Alarm SDK)
  • PTZ through ProServer (Untested)
  • Outbound audio from viewscape desktop

Pre-requisites

  • Viewscape Version 9.17 or greater
  • Veracity AdPro UserControl (This is required for Replay in Viewscape) and can be found on Dropbox in 'Technical/Deployment/AdPro/Fast Trace'.
  • Copy the zipped replay files into the root folder of Viewscape.
  • C++ Redistribute 2005 SP1 (2008?) -- For Pro server only (Alarms)
  • C++ Redistribute 2017 -- For surface and desktop

Customer Requirements

This section contains a list of requirements which need to be obtained from the customer to configure the integration.

NVR details

The customer or installer must be able to provide the following for each AdPro NVR on the system.

  • IP4 Address
  • User name
  • Password

Camera Channel Information

The required information for cameras in Viewscape is as follows;

  • Camera channel index

This channel index must match the NVR channel index used in the NVR configuration in Viewscape.

This index can easily be found in the camera configuration screen in the AdPro configuration software.

Installation

Viewscape

  • Install the AdProReplay UC by copying to root of vtas and running register.bat. This is the component that Viewscape loads for outbound audio.

NOTE: This must be installed into the same directory as Viewscape because the SDK runs an external executable (called AdproXOaNetSource.exe) to establish network communications with the device. It needs this to be in the same working directory as the host application unfortunately, otherwise the SDK can't find it.

  • Install Microsoft Visual C++ 2017 (x86) (included in zip)

Adding an AdPro camera

  • Create a camera with the AdPro camera type.
  • Ensure the camera is added to an NVR on the channel that corresponds to the same channel index used on the AdPro unit.

Surface

  • Copy the contents of the 'Depends\Adpro copy' files into the root folder of StreamSurface. The AdproNetSource is executed in isolation and used for all network communication by the SDK. It has to be in the same path as StreamSurface for live video to work.
  • Install Microsoft Visual C++ 2017 (x86)

Pro Server

  • Copy entire contents of the FastTrace2 SDK files into the root folder of pro server. These are stored in Depends\FastTrace2\2.02.111 copy
  • Install Microsoft Visual C++ 2005 Service Pack 1 Redistributable Package ATL Security Update for alarms
  • Set 'VSKAlarms' to True in pro server settings. (IF NOT USING ADPRO ALARM SERVICE)
  • Debug log entries appear as 'V3100'
  • Make sure the "ViewscapeLinkRemotePort" setting (Settings Editor) is matching to what's set in the VS Adpro Alarm Service.

Alarms

XO Config

  1. Launch the Ad Pro XO software (can be downloaded by browsing to the NVR webpage)
  2. The AdPro device connects to pro server to send alarms so requires configuration in the alarm forwarding section, where you can add up to 5 different IP address for alarms to be dispatched to.
  3. Ensure that the IP address for Pro server is correctly configure.
  4. Each alarm type needs its behaviour configuring to enable alarm forwarding. This is annoying but necessary. Refer to the AdPro installation manual for more detailed instructions on this. Generally, if an alarm is set to "Profile -- Normal Event" then the alarms will be received by Pro Server (Also check alarm schedules)

Obtaining Alarm Info

  1. Use Adpro XO to gain info about alarms and to set them up
  2. There is a corresponding Alarm Point Number for each type of alarm, these can be found in Adpro XO in the "System -- Behaviour -- Input/Output Behaviour" menu. The Device ID (Server Unit ID in Adpro XO) can be found and changed via the "System -- Alarm Transmission -- CMS Alarm Transmission" Menu
  3. Set up a Viewscape 'Adpro' device and enter the matching Device ID and one or more Alarms with the corresponding Alarm Point Number.

Troubleshooting

Common Issues

  • Replay does not work - video is black
    • We had issues with this when testing in the office with Version 4 of the SDK. The firmware on the device had to be updated to version 4.02 in order to resolve.
  • Adpro Playback error -9000 when using playback form and selecting a camera
    • Known issue when video cannot be displayed from the recorder. Found to be resolved when synchronising the time of the DVR. Enter the time manually and ensure it is within 5 seconds of the PC time
  • Ad Pro Playback error -910 when using playback form and selecting a camera
    • Known issue when video is unavailable for that channel. Could be a channel issue or a recording issue
  • Export only works once, then fails on subsequent attempts.
    • This is an issue in versions of the user control prior to 1.2.0 Install the latest user control to resolve.
  • Error when enabling VSK alarms "Message : Unable to load DLL 'AlarmServer.dll': The application has failed to start because its side-by-side configuration is incorrect.". This is because the server is missing C++ redistribute 2005 on the server (pre-requisite). May need to use AIO Runtime.
