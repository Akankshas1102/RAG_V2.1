Overview

The purpose of this document is to provide detailed instructions for the integration of CSL Dualcom into Viewscape.

Requirements

Functionality

Viewscape is able to handle incoming alarm inputs providing the devices and required pro events are configured.

Pre-requisites

  • Viewscape Version 9.29.0 or greater
  • Pro Controls 2.8.1 or greater

Customer Requirements

This section contains a list of requirements which need to be obtained from the customer to configure the integration. In order to configure CSL Dualcom into the Viewscape system, the following information is required;

  • Server IP Address
  • Pro events list for each alarm input

Installation

Viewscape Configuration

Adding a CSL Dualcom/WebwayoneRCT Server

  • WebwayoneRCT Server device is added in the following menu:

Admin > Command&Control > Device State Routing > Device

  • Once the device is added, Pro Server needs to be restarted.

WebWayOneRCT Server

  • Enter a name for the WebwayoneRCT Server and select the Device Type "WebWayOneRCT".
  • Enter the Server address in the format:
    • <ipaddress>
  • To specify a different port to the default enter
    • <ipaddress>:<port>
  • Click Save
  • In Setting Editor under ProServer Change "DebugWebWayOne" to True

  • Reboot Pro Server. When restarted you should see the following information in Pro Server Logs

Adding WebWayOne SIA Zone

  • WebWayOne SIA Zone device is added in the following menu

Admin > Command&Control > Device State Routing > Device

  • Give the SIA Zone a name and select the Device Type "WebWayOne SIA Zone"
  • RCT drop down box select the WebWayOneRCT Server you added previously
  • To get the Installation Number and Zone you will need to trigger an input on the CSL Dualcom Pro input device. Trigger input 1 by shorting contact from RTN and 1 contacts on Dualcom Pro Device
