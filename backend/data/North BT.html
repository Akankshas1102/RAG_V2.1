Overview

North Building Technologies controllers link systems together to optimise building performance and simplify building management. The controllers contain driver technology which can access the data within thousands of other products. They also have drivers to allow third party systems to synchronise with and adjust information on the North installation.

Functionality

  • Trigger pro events using an interface to North's controllers Essential Data (see 'Commander Tutorial.pdf' page 27 for overview of Essential Data). When values in Essential Data change, North's controllers send a signal to vtas with details of the values that have changed. Viewscape then checks if a condition is met and fires a pro event.

Pre-requisites

  • North's controllers must have the DataSync driver installed. This may require an additional license (see 'Commander Tutorial.pdf' page 18).
  • Leave DataSync driver settings as default. Port should be 1920.
  • Essential Data should be configured (see 'Commander Tutorial.pdf' page 27).

Installation and Configuration

Pro Server

No configuration required. If adding or editing North Controllers, pro server must be restarted.

Viewscape

  • Add North Controllers using Admin->VTAS Pro->NorthBT Controller:
    • 'Object ID' must match the corresponding Essential Data object reference (see 'DataSync Driver Manual.pdf' page 6).

      NOTE: this is the Datasync oid not the Essential Data object reference.

    • 'Condition For Alarm' is the test for a positive alarm. This can include the previous value vtas saw. Any logical expression is allowed ie ((PREV\<10\*2) AND ((NEW=1) OR (NEW\>5)))

Troubleshooting

  • Use North's ObSys software to view, setup and debug Essential Data before troubleshooting issues with Viewscape.

Debugging

  • Communication between pro server and the North controller can be viewed in the log with prefix (NorthClient) after setting VTASProServer.exe.config->DebugNorthBT=True.
