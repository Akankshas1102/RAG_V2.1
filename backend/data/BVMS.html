Overview

This document outlines Viewscape integration with BVMS.

Functionality

  • Live/Playback/Clips
  • PTZ
  • Events

Live/Playback/Clips {#live}

Pre-requisites {#livepre}

  • BVMS server accessible by viewscape clients.
  • Matching version of BVMS installer to BVMS server ie 11.1.

Installation {#liveinstall}

  • Fire up the BVMS installer on the Viewscape clients and install 'Cameo SDK':

  • Check the CameoSdk is installed at default location 'C:\Program Files\Bosch\VMS\bin'. If not alter the Surface setting 'BoschCameoSDKPath' to match accordingly. Leave blank for default path.

Usage {#liveusage}

  • To configure the BVMS channels in Viewscape, match Viewscape nvr channel 'Display Name' with the 'Logical Number' in the Bosch Configuration Client (or the CameoSdk Sample (installed with the CameoSdk)). Note: you can also use the Bosch 'Display Name', but be wary of duplicate names:

PTZ {#ptz}

PTZ requires a seperate relay application to relay the commands between ProServer and the BVMS server.

Pre-requisites {#ptzpre}

  • BVMS server accessible from the relay app, usually on the same machine as ProServer.
  • Matching version of BVMS installer to BVMS server ie 11.1.

Installation {#ptzinstall}

  • Copy the VSBVMSPtzService folder to the desired machine for relaying the ptz, preferably the same machine as ProServer.
  • Fire up the BVMS installer on the relay app machine and install 'Cameo SDK':

  • Open VSBVMSPtzService.exe.config to setup the relay:

    • Endpoint, Port
      • Listening IP and port for relay app, for ProServer to connect to. Default is localhost, 51051
    • BoschSDKBinFolder
      • Check the CameoSdk is installed at default location 'C:\Program Files\Bosch\VMS\bin'. If not alter to match accordingly.
    • BVMSServer, BVMSUser, BVMSPassword
      • BVMS server IP and credentials.

  • Alter ProServer setting 'BVMSPTZServerTarget' to the IP and port of the relay app ie 'localhost:51051'

Usage {#ptzusage}

  • The camera in Viewscape must have 'Use NVR For Live Playback' ticked.
  • Start VSBVMSPtzService.exe and press 'Activate'
  • If successful, the log will show it is connected to the BVMS server and started the MCP service:

  • The app can now be left running while restarting ProServer.
  • Start ProServer as normal.

Events {#events}

Events require a seperate relay application to relay the events between the BVMS server and ProServer.

Pre-requisites {#eventspre}

  • BVMS server accessible from the relay app, usually on the same machine as ProServer.
  • Matching version of BVMS installer to BVMS server ie 11.1.
  • ProServer.ViewscapeLinkRemotePort setting valid.

Installation {#eventsinstall}

  • Copy the VSBVMSAlarmService folder to the desired machine for relaying the events, preferably the same machine as ProServer.
  • Fire up the BVMS installer on the relay app machine and install 'BVMS SDK':

  • Open VSBVMSAlarmService.exe.config to setup the relay:
    • MCPServer, MCPPort
      • ProServer IP and 'ViewscapeLinkRemotePort'
    • BVMSServer, BVMSUser, BVMSPassword
      • BVMS server IP and credentials.

Usage {#eventsusage}

  • Congifure 'BVMS Device' devices in Viewscape with 'Device Name' matching the device/event in BVMS:

  • and make sure the state 'Triggered' is configured:

  • Start VSBVMSAlarmService.exe and press 'Activate'
  • If successful, the log will show it is connected to the BVMS server:

  • It is advisable to restart this event relay app after ProServer is restarted.
