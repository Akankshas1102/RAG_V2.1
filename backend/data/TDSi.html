Prerequisites

Before you install EXgarde, ensure that you have done the following:

  • Restarted the machine.
  • Checked that ASP.Net is enabled in windows features on the target PC.
  • Ensure that .NET 2.0 or greater is installed on the target PC.

Requirements

  • To begin you need the EXgarde SDK address, it is usually something like this:
    • http://{hostnameofmachine}/Exgarde/
  • To access the config file for the SDK, navigate to
    • Program Files (x86)/TDSi/Exgarde/SDK/Runtime/Service
    • Look for Exgarde.Network.Service.exe.config. >IMPORTANT -- only modify this file if you have installed EXgarde on an existing SQL Server >Instance. If that is not the case, then I strongly suggest that >you do not modify the file. Refer to Figure F. There you will >see the web address for the SDK which will be useful for setting up >Viewscape.
  • For Viewscape Setup, make sure that in Viewscape Settings Editor V3.5 you have set the Exgarde address so that VTAS Pro Server can connect to the SDK. (Refer to Figure G)

Installation

Install EXgarde

IMPORTANT

Before you begin installation, ensure that you have created/or have a separate VM Server for installing Exgarde. Exgarde needs to be installed on a standalone server do not install it on a server that has ProServer running on it or on the same SQL Server Instance as SQL Server.

This is to reduce the level of confusion and the risk of configuration errors that might occur due to installing it on the same machine or existing SQL Server.

In this installation of the Exgarde, you will be installing the following applications:

License Server

This application is very crucial and needs to run in the background. (It is the Exgarde Server) and it has all the license information required for the Exgarde SDK. Without it you cannot run the SDK Service, and the rest of the Exgarde applications mentioned in this document.

EXgarde Communication

This app listens and fetches any incoming events coming from the ACU (Access Control Unit). It also displays the current state of the ACU whether it is on/off-line.

EXgarde Synchronizer

You will need this application to build the Fusion Database which is required by the SDK.

EXgarde Fusion You will need to configure and start a service

instance for Fusion so that the SDK can connect to the Fusion Database, which will update data in the EXgarde Database.

EXgarde Explorer

A Client Application that allows you to add and configure any specific objects/devices/personnel that are required in the EXgarde Software (i.e. ACUs, Keyholders, Key, Areas, Groups etc.).

IMPORTANT: Make sure that all the applications mentioned above are running (apart from EXgarde Explorer), you can run this if the customer wants to configure, new ACUs or Keyholders any configuration that is specific to Exgarde.

Step by Step installation

  1. Navigate into the folder called DISK 1 and run 'Setup.exe' in the root of the folder.
  2. During the installation process, you will be asked which product type of EXgarde you want to install. IMPORTANT Make sure you select Exgarde Professional because we have been issued a professional license key.
  3. Then when asked about the Installation Type, Install Exgarde Server (This section is up to you, but I recommend choosing the first option which installs the SQL Server Express). Refer to Figure B.
  4. NOTE the next section all the applications you will need and the once I have mentioned will be ticked by default, do not untick any!
  5. The next page will show you the target directory where EXgarde will be installed. Leave them unchanged.
  6. Lastly, some personalisation like which application do you want to create a desktop icon for.
  7. Next page -- make sure you tick license server to be the first software that starts up.

Next Steps

Now that you have successfully installed EXgarde, proceed to launch the EXgarde Explorer software. Login using the following password: TDSi. Then click on the link to the Documentation Link and you will find documentation on every part of the application. I highly recommend that you read the ExSync Issue, EXgarde Fusion Issue and the EXgarde SDK Issue.

In addition, in Exgarde Explorer, when you press the F1 Key on the Keyboard, this will open the application help section, which has step by step instructions on how to set up the ACU etc.

ACU & Door Configuration Setup

In the EXgarde Explorer app, select ACU Wizard and follow the step-by-step guide and configure the ACU and the names and the number of doors, readers associated with the ACU.

Make sure whilst configuring the doors you give appropriate names for the doors (i.e. Door 1, Door 2 etc). Do the same for Inputs and Readers.

Refer to Figure A for step-by-step ACU Set Up.

IMPORTANT -- Remember to set the network address of the machine EXgarde is installed on, in the Settings Editor. Look for ExgardeAddress setting in the list of settings for Pro Server.

SDK Installation

Again, there is a step-by-step guide available in EXgarde Explorer but when you run the installation make sure (especially if you have installed with the option -- EXgarde Server -- Install SQL Server Express with Exgarde or use existing SQL Express), you will have default credentials for the SDK. The SDK will use those credentials to connect so do not change it. ONLY change the credentials if you have done a custom installation of EXgarde.

Before you begin this configuration process, make sure that you have completed ExSync Synchronisation process, by building a new Fusion Database. Get the connect string from the Fusion Database.

  1. Configure the Service -- note that the connection string may be slightly different.

  1. Once you have configured everything press the test button to ensure that the connection string is correct, and it connects to Fusion DB.

  1. Double click on the service to open the Service Details dialog and start the service.

Running the SDK

  1. You can open Windows Services and look for the Service EXgarde SDK the status is usually Automatic, but it is advisory to set to manual so that you can ensure license server is running before you run the service.
  2. Alternatively, navigate using File Explorer/Command Prompt to the following directory: Exgarde/SDK/Runtime/Service/ with the following command -- exgarde.network.service /console

For option number two you should see the SDK logs in the console, and whether Pro Server was successful in connecting to the server.

Viewscape Device Setup

NOTE that the TDSi Version 9 integration will only work on Viewscape version 9.22 or higher. Similarly, Pro Server needs to be on version 9.22.0 or higher. Also, ensure you have the latest version of Viewscape Settings Editor installed.

In Viewscape, begin by setting up the relevant TDSi Devices (They will usually be TDSi Door, Input and ACU (Access Control Unit)). Also, the TDSi Device Configuration on Viewscape is quite simple you can configure the device as stand-alone devices rather than parent and child devices.

When you are adding a new TDSi Device (in this example we are adding a Door), ensure that the Door ID is the same name as the name of the Door on Exgarde Explorer. This is because when events are triggered incoming events are only handled in Viewscape when the Device ID matches the name of the Device Configured in Exgarde Explorer.

Viewscape currently supports all the events for Doors, Inputs and ACU. Some of the events cannot be tested due because they cannot be triggered manually. Examples of the following ACU events: Progress, System Fault, Mains Power On/Off Event. However, events such as Reader Box Faulty and Reader Box Fixed can be generated manually.

Here is a list of Device Events that can be tested:

Door Events:

  • Access Granted
  • Access Denied
  • Invalid Key
  • Egress On/Off
  • Door Open/ Closed
  • Door Forced
  • Door Ajar

ACU Events:

  • Timeout
  • Case Open/Closed
  • Return from Timeout
  • Mains Power Off/On
  • Time Error
  • Progress

Input Events:

  • Input On/Off

VTAS Pro Live

In VTAS Pro Live, once all the TDSi devices and it is events have been configured, when you trigger the event, you should expect to see the alert listed in the Alerts window. Refer to Figure D in Appendix.

Similarly, in Pro Live, in the Device List window, you can view all the TDSi Devices that have been configured in Viewscape and you can invoke actions, which are specific to that device. Refer to Figure E.

Troubleshooting

  1. If you are testing the events system and you do not see any events appearing on the software or the SDK Webpage, ensure that you have EXgarde Communications running. Without this app, you cannot receive any live event messages.
  2. If you are having trouble connection to the SDK check if the SDK is running as a service in the first place.
    • To run the SDK such a way that one can see the logging information (for testing/debugging purposes), run the following command in Command Prompt which needs to point to the following Directory:
      • Exgarde/SDK/Runtime/Service/
    • With the following command :
      • exgarde.network.service /console (Refer to Figure C)
  3. SDK is unable to run...
    • Make sure you are connected by Starting the License Server and make sure that you have created a Fusion Service and it is running in EXgarde Fusion.
  4. If Fusion Service is unable to start and displays status:
    • "ERROR - RETRY"
    • Make sure you use the ExSync Synchronisation tool to purgethe database, and try again.
