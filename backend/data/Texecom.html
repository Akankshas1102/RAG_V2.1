Overview

The Texecom Panel integration uses the Viewscape Drive API, this API allows the integration to be installed, updated, and run separately from the Pro Server.

Functionality

The application can send state changes for the Texecom Panel as well as its areas and zones.

The following are possible states for Panel:

  • Offline
  • Invalid Configuration
  • Active

The following are possible states for Areas:

  • Unknown
  • Disarmed
  • In Exit
  • In Entry
  • Armed
  • Part Armed
  • In Alarm

The following are possible states for Zones:

  • Secure
  • Active
  • Tamper
  • Short
  • Zone In Fault
  • Failed Test
  • Alarmed
  • Manual Bypassed
  • Auto Bypassed
  • Zone Masked

It should be noted that - Zones can be in multiple states at once i.e. **Tamper, Alarmed** would occur if the zone had been tampered while the area is in alarm state

The latest release does NOT support command and control i.e. VIEWSCAPE asking to arm an area. This will be added in a future release.

Pre-requisites

  • The panel requires a PC-COM interface module which provides a serial port to connect to the panel.
  • This serial port must be connected to the Texecom Driver.
  • Pro Events must be already configured.
  • Texecom Panel only allows 1 TCP connection at any given time. If the panel is configured to speak to a Texecom Connect device or other app using SmartCom then it will conflict with our application. It is therefore critical to ensure that when using a texecom panel that no other devices are trying to communicate with it over TCP.

Texecom Panel Configuration

  • The texecom panel must be programmed correctly to allow the application to speak to it.
  • It should be noted that to function optimally the following applies:
    • All Zones should be given a name. Without unique names the app will default to naming the zones zone1 etc. These names are given to VIEWSCAPE for display purposes.
    • All Areas should be given a name. Without unique names the app will default to naming the areas area1 etc. These names are given to VIEWSCAPE for display purposes.

Com Port Setup

  • On panel enter engineer code
  • Press 7 on the keypad(UDL/Digi Options)
  • Press Yes
  • Press 8 on the keypad(Com Port Setup)
  • Press Yes
  • Ensure Com Port 2 is set to ComIp Module
  • Ensure all other Com ports are set to Nothing Fitted

UDL Password

A UDL password MUST be programmed.

  • On panel enter engineer code
  • Press 7 on the keypad(UDL/Digi Options)
  • Press Yes
  • Press 5 on the keypad (UDL Options)
  • Press Yes
  • Press 4 on the keypad(UDL Password is:)
  • Press No
  • Enter a password and press Yes

ARC Access

ARC access MUST be disabled

  • On panel enter engineer code
  • Press 7 on the keypad(UDL/Digi Options)
  • Press Yes
  • Press 3 on the keypad(Program Digi)
  • Press Yes
  • Ensure ARC 1 through number of ARC channels are all set to Disabled

Omitable zones

Zones MUST be configured to be omitable

  • On panel enter engineer code
  • Press Yes(Zone Setup)
  • Press Yes
  • Press No
  • Press 1(an O should appear in the attribs)
  • Press Yes
  • Go through the above steps for each zone

On Completion

  1. Check you have 1 COM Port set to the ComIP module and no others are connected.
  2. You have a UDL password.
  3. ARC access is disabled.
  4. Ensure zones that can be omitted are configured correctly

Default panel passwords:

  • 1234 engineer
  • 5678 master

Installation.

  1. Unzip the folder
  2. Open a powershell as administrator and navigate to the location where you unzipped the folder

NOTE - This location matters and must not change after the service is started otherwise it will cause issues finding required files

  1. Run the install.ps1 file with the following arguments
    • -IPAddress "ip_address" -Port port
    • i.e. .Install.ps1 -IPAddress 10.1.1.191 -Port 12345

NOTE This file may be blocked. You may need to right click and go to properties and mark file as unblocked before running it

  1. A service with the name of TexecomViewscapeService should now be shown under the Windows Services menu
  2. Click the start button to start the service

Uninstall

  1. Navigate to the service in Windows Services menu and stop theservice
  2. Open a powershell as administrator and navigate to the locationwhere you unzipped the folder
  3. Run the Uninstall.ps1 file. This will uninstall and unregister the service

NOTE This file may be blocked. You may need to right click and go to properties and mark file as unblocked before running it

Configuration

All service configuration is done automatically when the service runs the first time. It will create a configuration file called:

  • IdMapping.json In the following location:
  • %localappdata%/Veracity UK Ltd/SUPL/Viewscape/TexecomIntegration OR if running service as system:
  • System32\\config\\systemprofile\\AppData\\Local\\Veracity UK Ltd\\SUPL\\Viewscape\\TexecomIntegration

This file should not need modified but should not be removed after first run as it is used to correctly map the IDs of the service with VIEWSCAPE.

Logging

To view the logs for the application we can use Event Viewer on Windows

  1. Open Event Viewer
  2. Select Applications and Services Logs
  3. Select TexecomViewscapeService

Viewscape Configuration

Entire section to be updated, below from legacy setup.

When configuring the panel within Viewscape, there are two things to consider:

  1. Panel alerts
  2. Zone alerts
  • Each has its own set of alerts ie the panel may report a comms failure, or a zone may report an activation.
  • The panel is identified by the serial port.
  • The zones are identified by the panel internal zone number.

Panel Alerts

  • Configure each panel via
    • Admin -> VTAS Pro -> Alarm Panel -> Device
  • Choose 'Texecom Panel' as the Device Type:
  • 'Server Com Port' is the serial port that the panel connects to on the Viewscape server.
  • 'Password' is the UDL password setup on the panel.
  • One or more panel events can be added using the 'Texecom Panel Event' drop down, then linked to a pro event.

Zone Alerts

  • Configure each panel zones using the same form:
    • Admin-> VTAS Pro -> Alarm Panel -> Device
  • Choose 'Texecom Zone' as the Device Type:
  • Texecom Panel links back to the panel record created above.
  • Texecom Zone is the zone number on the panel.
  • One or more zone events can be added using the Texecom Zone Event drop down, then linked to a pro event.
  • Note: disregard the Event Codes number as this should match the Texecom Zone Event wording, but shows the internal reference.
  • See Alarm Device Form manual for additional settings.
