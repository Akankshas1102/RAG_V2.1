Overview

The i-comply keyboard is a simple keyboard device. It is a physically robust device, designed to perform the most commonly used control functions within VTAS. The keyboard is designed to communicate via one of the following methods;

  • RS232
  • USB
  • Ethernet

The keyboard is IP based and a network cable must be used and is shipped without any cables, therefore it's assumed the site will provide a PoE (Power over Ethernet) switch. If no PoE Switch is available, a USB-A to USB-B cable may be used in conjunction with the standard network cable, however this is not advised due to the power requirements.

Functionality

  • LCD display for camera control information
  • Joystick for PTZ control
  • Fully customisable keys for VTAS

Pre-requisites

  • Surface Controller

Installation

  1. Connect the keyboard to the network
  2. Configure the keyboard
  3. Configure Surface Controller

Keyboard Setup

The configuration menu for the keyboard is entered by holding the F2 and 0 buttons down while there is no power to the keyboard, then restore power to the keyboard while still holding down the keys. Once the menu is displayed, the buttons carry out the following functions:

  • F1
    • Skips to the next option in the menu
  • F2
    • Confirms the entry for that option
  • 1,2,3,4 keys
    • Move to the next corresponding value within that option
  • Joystick Up / Down
    • Moves the values up or down
  • F4
    • Exits out of the menu

It's important to note once having configured the IP address of the unit, it may take around 30 seconds for the unit to respond on the network.

The required configuration is as follows;

  • Kbrd 192.168.0.80
    • Set the IP address of the keyboard
  • Kb TCP Port 002.188
    • Set the port number in binary (default: 700)
  • Srvr 192.168.0.81
    • Set the IP address of the Workstation or Client connected

Surface Configuration

The configuration in surface is very similar to that of the AXIS joystick, however the IP address needs adding, the keyboard type value is different, and the function mappings use the prefix 'z'. Configure the SurfaceController.exe.config as follows;

<setting name="KeyboardIPAddress" serializeAs="String">
    <value>172.16.10.209:700</value>
</setting>

<setting name="KeyboardType" serializeAs="String">
    <value>64</value>
</setting>

Micro Client

Micro Client is an intel NUC with surface controller on it which the keyboard connects to, and allows the user to login and control cameras without using the VTAS application.

To configure this, a button must be configured on the keyboard to login (PinLogin id = 23).

Using this method, the user is unable to change the layout of surface, and there the layout is configured in the SurfaceController.exe.config file, in the section:

<setting name="SurfaceLayout" serializeAs="String">
    <value>1</value>
</setting>

For more layout options, see appendix section "Surface Layouts".

To allow the user to login, a "Keypad PIN" number is required for the user account in VTAS. This is done from the Admin -> User menu, select the user, and then set in the pin on the "Pro" tab (Figure 1).

Figure 1: Keypad PIN

Users should use 4 digits or more for the keypad PIN, and they can login once configured, by entering the PIN, followed by the login button configured in the initial step. To logout, use PIN "0000", followed by the login button.

Support

The next section describes the process of troubleshooting issues with the keyboard and any debugging that may be required.

Troubleshooting

Due to the network port on the keyboard being slow to respond initially, it's important to leave it 30 seconds before launching surface controller. Also ensure the IP address responds to ping commands.

  1. When the keyboard first powers on it will display a firmware version. Please ensure this is the latest provided
  2. Once the unit is ready, it will display "WAIT" on the LCD screen
  3. If the Surface is connected to the keyboard, the LCD screen will display "Awaiting Login" (for VTAS)
  4. If Surface Controller cannot communicate with the keyboard, an error message will display "Cannot connect to IP"

Debugging

  • VTAS Pro Server should show joystick commands from the client, the same way as using the target within Pro Live, Joystick tab
  • Debug tool located in the same directory as the manual

Upgrading the Firmware

The upgrade procedure is carried out using the downloader2 software which enables the firmware to be uploaded to the keyboard. In addition to this, the MCP2200 driver must be downloaded from the microchip.com website and installed once the keyboard is connected. Although limited amount of time has been given to testing the upgrade procedure, it has been noted that the driver doesn't work on Windows 8 operating systems, and Windows 7 or XP should be used if that's found to be the case.

The required files should be available on the NAS in the same location as this document.

Pre-requisites

  • Jumper switch
  • USB-A to USB-B Cable
  • DC 12V Power Adapter
  • MCP2200 Driver
  • Downloader2 software

Procedure

  1. Connect the jumper switch vertically to the left side facing the unit (Figure 2 Appendix)
  2. Connect the USB cable and power adapter
  3. Find the COM port from device manager
  4. Launch the downloader2 software and select the Comms port
  5. Browse to the firmware using the folders in "Source Directory"
  6. Click contact target and check the result confirms Comms is established
  7. Select the firmware in the "Source File" section and click "Download"
  8. Remove the jumper and power cycle the unit
