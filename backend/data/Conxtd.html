Overview

This document outlines the features Viewscape supports for integrating with Conxtd to accept events from their cloud based service.

Functionality

  • Listen and accept new events from edge devices (ie dualcom) routed through the Conxtd network forwarded to Viewscape.

Pre-requisites

  • Publicly accessible web domain, with valid SSL certificate https authentication.
  • fullchain.pem and privkey.pem SSL certificate files from the web domain.
  • Port fowarding of https (tcp 443) to Viewscape Conxtd web server app.
  • .Net Core 6 for Viewscape Conxtd web server app.
  • Open connection from Viewscape Conxtd web server app to Viescape Server ViewscapeLinkRemotePort.

Installation

A separate 'Viewscape Conxtd web server app' is required to be installed on a suitable machine (server side).

Conxtd Web Server

  • The Viewscape Conxtd web server app (folder copy/no installer needed) will need to be installed on the machine where the domain https port forwarding is directed to. The Viewscape Conxtd web server can be found under Deployment/Integrations/Conxtd [citation needed].
  • Configure the appsettings.json file within the web app folder using a suitable text editor:
    • MCPAddress, MCPPort
      • Viewscape Server ViewscapeLinkRemotePort details.
    • UseUrls
      • This defines what port the web app is listening on for https (and http, but not required). Change this to the port forwarded from the public web domain (ie public web domain https port 443 ported to web server app port 5004)
    • Kestrel Certificates
      • File paths to the fullchain.pem and privkey.pem SSL certificates for the web domain.
  • Run VSConxtd.exe

Normal running example.

Viewscape Server

ProServer.ViewscapeLinkRemotePort setting valid.

Usage

Devices in Viewscape need configuring to accept specific Conxtd events. There are two devices available for Conxtd in Viewscape:

Conxtd Site

Each Conxtd site must have a separate device record in Viewscape. Site ID corresponds to the 'unique_site_id' from Conxtd.

Conxtd Event

Each Conxtd site may have one or more events. Each Conxtd event must have a separate device record in Viewscape. Event ID corresponds to the 'unique_event_id' from Conxtd, and Parent Site is the corresponding parent Conxtd site device.

Support

The Viewscape Conxtd web server app is a console based app that shows information on screen, but there is a also a Log.txt generated in the root folder with additional details and error information.
