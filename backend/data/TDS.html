Overview

Pre-requisites

  • Viewscape version 9.2+

Client Requirements

  • TDS Server Address
    • The machine with Corinio running, they have been confused by "Server" as the terminology for this.
  • Username and Password for SDK access.
    • Known as Winguard details in the Corinio config
  • Pro server must be able to ping TDS Server
  • Event, Door, Controller and Contact codes
    • Can be retrieved from the Devices tab of Corinio

Installation

Server

To enable functionality in Viewscape at least one server must be configured as a device.

To do this create a device with type of "TDS Server" and enter the required details. The address should be formatted as follows:

IP.IP.IP.IP:PORT

Doors and Controllers

There are three different types of devices in the TDS system; Doors, Readers and Controllers. In Viewscape the former two have been combined into one type (Doors) as Readers are always linked to one door.

Both Doors and Controller devices have a set of hardcoded states as well as custom events. The custom events are for what the SDK defines as a "Booking Event" which are issues with the door/reader operation, such as unauthorised errors. To define a custom event, you will need the event code which will need to be defined in the "Additional Event Code" in the Device setup form. These events are in no way hard coded and will differ entirely from site to site, so they will need to be acquired from the client.

Contacts

Contacts are similar in operation to the other types of devices in the TDS system, except they have no pre-defined states, and instead of booking events they fire "Contact Events". These events work in the exact same way as booking events.

NOTE: - Both Booking and Contact event codes are not unique, and different devices could potentially use the same code to mean different things, requiring us to use individual State Routings to keep them separate.

Debugging

Enabling DebugTDS in pro server config will make the integration extremely verbose, and should definitely be turned off in standard use.
