Overview

This manual contains the required information and steps taken to integrate a Vemotion Streaming Server with Viewscape.

"The Vemotion Streaming Server (VSS) is the hub of the Vemotion solution and connects the Encoders and Viewers to the Network Operations Centre (NOC)". The VSS is a software application used to integrate cameras and encoders to an unlimited number of licensed viewers. The VSS must have internet access to retrieve the wireless video from the cameras and encoders, and 3G and 4G connections are widely used. The VSS is licensed to allow a specific number of remote encoders.

The VSS also authenticates users that connect to it and as such are listed within the software when connections are established and maintained. Because of this, a new user is required for the integration with Viewscape.

Functionality

  • VSS Allows for streaming of wireless cameras and therefore doesn't support video playback

Pre-requisites

  • Microsoft Visual C++ 2008 Redistributable
  • Vemotion SDK

Installation

The VSS is installed on a Windows based server and network connectivity should be checked prior to configuration with VTAS, as well as ensuring a user is created for Viewscape.

Before taking any steps listed in this manual, verify the Vemotion integration is not already installed by checking the existence of the Vemotion 'Pro Camera Type' from the information menu and then select 'Camera'.

The installation of the Vemotion SDK is to be installed on all VTAS clients which have the requirement of streaming video from the VSS and is not required on the VTAS Server.

The Vemotion installation comprises of the following steps;

  1. Install Vemotion SDK
  2. Update Viewscape License
  3. Update Viewscape Server Config file with Vemotion license key
  4. Update the configuration file of VTAS Pro.exe or StreamSurface.exe
  5. Configure cameras in Viewscape

The details of each step can be found in the relative sections below.

Software

The required software for Vemotion are simple wizards for Microsoft Visual C++ 2008 redistributable if not already installed, and the current version of Vemotion SDK. There are no further details required for these installations.

Viewscape Configuration

Updating the Licence

Update the current license key from 'Help' -> 'About VTAS' (Provide developer with current key and obtain the new key)

To enable the PTZ functions to work:

  • edit the Server Config located in: C:\\Program Files (x86)\\i-comply\\VTAS Pro Server\\VTASProServer.exe.config
  • Find the setting VemotionServerLicence
  • Within the value tags, enter the PTZ licence key obtained from Vemotion
    • for example: icomplyptzlicence

Camera Configuration

To view live video from the Vemotion cameras on the clients, the configuration file needs changing for VTAS Pro or the SurfaceController, depending on which is being used. The value is calculated to allow for the different interfaces each site has and therefore is likely to differ per site. The value for Vemotion only is '268435456', if you don't know the value required for more interfaces as well as the Vemotion please contact the author of this document.

For VTAS Pro, the configuration file is located in: ```c:\Program Files (x86)\i-Comply\VTAS Pro\VTASPro.exe.config`` Locate the setting Streamviewer and change the value to 268435456.

For the SurfaceController, the configuration file is located in: c:\\Program Files (x86)\\i-Comply\\StreamSurface\\StreamSurface.exe.config Locate the setting RestrictionsLive and change the value to 268435456.

Ensure a user account is configured on the Vemotion Streaming Server software and create this in VTAS by going to Admin -> Lookups, then select Pro Camera -- Stream Authentication.

Configure the cameras from the Camera menu located in Information -> Camera. Select the Pro Camera Type as Vemotion. Set the Pro Index as the unique camera number and aspect ratio as required. Enter the IP address and port number of the Vemotion Server in the relative field, for example 10.10.10.225:4444 (4444 being the default). Thenset the Stream Name as the name of the camera on the Vemotion server. Select the authentication in the Vemotion Stream Authentication field which was created earlier.

Support

Troubleshooting

Common issues and steps be taken are;

  1. If there is no option for Vemotion in the camera type, restart VTAS and verify the licence key entered is correct
  2. If the video won't display and reports it can't connect with the camera, check network connectivity to the VSS / camera authentication / camera stream name / video exists in Vemotion software
  3. If no PTZ is available, check the logs on the server to see if telemetry is being sent out, if not check the camera configuration and PTZ licence in the server config. If the server is sending the required telemetry, contact Vemotion
