Overview

The scope of this document is to provide instructions on performing a CTO (Configure To Order) on the Coldstore NVR's. This should be carried out prior to dispatch from the office.

Pre-requisites

  • Sitescape Version 3.0.771

Important

  • When changing the target DO NOT reboot the unit before adding it to Sitescape and updating the firmware. Otherwise this will prevent the unit from booting correctly
  • All cameras must have a stream profile configured (Axis) and include NTP configuration. Incorrectly set time and dates can cause recording and playback issues
  • Configuring various network and NTP settings using sitescape may not work, and in this instance it should be configured manually using instructions from section 4.1 Linux Commands in this document
  • When using Sitescape, specify the "type" when adding a device. Please note when upgrading from Coldstore to Coldstore NVR, this type requires changing as well as the password. Remove the Coldstore and re-add it to Sitescape.

Installation

The basic steps for ensuring the Coldstore is functional before dispatch are;

  1. Update BIOS Firmware
  2. If using Coldstore NVR -- change the firmware target using putty
  3. Add the Coldstore to sitescape
  4. Update Coldstore firmware
  5. Insert disks and verify (If no disks are provided by the customer, use test disks)

Update BIOS Firmware

  1. Press F2 or DEL to boot into the BIOS screen (Password 3816). If you are not prompted for a password, this would indicate that the BIOS has reset, and the CMOS battery has failed. If this is the case, please contact engineering for guidance. The version is indicated on the main page as "UEFI Version" and this MUST be P1.70H.
  2. Copy the BIOS firmware to a USB key from (D:\Dropbox (icomply Ltd)\Technical\Deployment\4 Integrations\Veracity\Coldstore\Firmware\CSTORE 15-3U PRO\BIOS Firmware), and plug the key into the Coldstore.
  3. In the BIOS, go to the Advanced tab and select the Instant Flash option.
  4. Select the \IMB1541\IMB1541.70h (P1.70H) and press enter. The BIOS update should then be complete

Verifying BIOS Settings

The BIOS settings should be verified first in case there are any issues retaining these settings, for example if the CMOS battery has failed. Boot the unit into BIOS with the F2 or DEL key. If a password is requested there's a good chance the unit is retaining it's BIOS settings, however type "3816".

From here go to Storage Options -> Advanced Tab -> Storage Configuration -> Select SATA 3_1 and SATA 3_2, and check Hot Plug is enabled for drives.

Changing the firmware target

Use putty to create an SSH session to the Coldstore NVR, and login using the standard credentials (root).

  • From the command line, then enter "cd /etc", followed by "ls", which will display a list of files. Verify the file "COLDSTORE_PRO_TARGET" exists.

    • If upgrading to Coldstore Version 2 no further action is required.

  • If upgrading to Coldstore NVR Version 3, the target file needs to be renamed, following these further instructions. This is to prevent customers from running incorrect upgrade files.

  • To change the name of the target file, enter;

"mv COLDSTORE_PRO_TARGET NVR_COLDSTORE_PRO_TARGET"

Adding disks

After all the previous steps are complete, insert the disks. If the customer isn't providing any disks, use the test disks. Ensure all drive bays are tested and working.

It's important to remove these disks afterwards and ship them separately to the unit.

Linux Commands

CommandDescription
IInsert command
EscLeave editor mode
:Prefix for x and q! commands
XSave and quit from editor
q!Quit without saving from editor

Changing the IP address

vi /etc/network/interfaces

Restart network services

If device is unreachable on the network (Plug in monitor to device)

/etc/init.d/networking restart

Check timezone

ls -l /etc/localtime

lrwxrwxrwx 1 root root 33 May 14 2018 /etc/localtime -> /usr/share/zoneinfo/Europe/London

Change timezone

rm /etc/localtime ; ln -s /usr/share/zoneinfo/Europe/London /etc/localtime

Change NTP Servers

vi /etc/ntp.conf

Restart NTP service

To allow it to sync (WARNING: Stops recording and unit requires rebooting to work again)

/etc/init.d/ntpd restart

Display NTP status

ntpq -p

Restart NVR Process is not running

To start it: monit start coldstore_nvr_app

To see what's running: monit status
