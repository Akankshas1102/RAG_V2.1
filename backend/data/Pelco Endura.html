Overview

  • Pelco Endura SDK covers the use of their Endura recorders and cameras. This is essentially everything but the old DX8100 and DX8000 recorders and the Pelco Digital Sentry range.
  • The Endura environment is a powerful sub system of components that auto detect each other and are usually all managed by a central machine called a 'System Manager'. If they are using a system manager then these details are required. Sometimes we simply need to access some Pelco Spectra cameras that are standalone, in which case there will be no system manager.

Functionality

  • Live / Playback / PTZ / Export

Pre-requisites / Software

  • For video clients (surface or VTAS) you will need to install
    • Directx 9c Jun2010 redistributable. (directx_Jun2010_redist.exe)
  • For all video clients and Server you need to install
    • Pelco SDK 3.3.1 (PelcoSDK-3.3.1-10524.vc9.exe)
  • For VTAS replay you need to install out user control component
    • EnduraUCSetup.msi or .exe

Customer Requirements

  • Where applicable for System Manager: IP, port, username, password.
  • Standalone cameras: Camera Name, IP, port, username, password.
  • NVRs: NVR name, IP, port, username, password.
  • NVR Channels: Camera Name, Pelco UUID.

NOTE: NVR channels can be added in any order because they use the Pelco UUID and not the Channel number to connect.

Installation

  • VTAS Server:
    • Install the Pelco SDK 3.3.1
  • Surface and VTAS Clients:
    • Install Directx 9c Jun2010 redist.exe, it'll extract into a folder you specify, locate and execute the Setup.exe file.
    • Install Pelco SDK 3.3.1 (PelcoSDK-3.3.1-10524.vc9.exe)
      • Modify the Windows Environment Variable "Path". And find the path to the SDK installation folder and replace 'Debug' with 'Release'.
      • You may need to reboot the PC.
      • COM Register the file C:\\Program Files(x86)\\Pelco\\API\\Libs\\Release\\PelcoAPICOMViewer.ocx\
  • Additional for VTAS Clients
    • Install playback control (EnduraUCSetup.exe/msi)

VTAS Configuration

  • Enable Pelco Endura in licence key.
  • Enable Pelco Endura in Surface.
  • In SurfaceController.exe.config:
<setting name="PelcoEnduraRTPPortStart" serializeAs="String">
    <value>9000</value>
</setting>

NOTE: for a wonderwall with multiple screens/SurfaceControllers, this setting need to differ by 100 per screen. EG. Surface1=9000, Surface2=9100, Surface3=9200

  • In StreamSurface.exe.config:
<setting name="PelcoEnduraPluginDir" serializeAs="String">
    <value>C:\\Program Files (x86)\\Pelco\\API\\Libs\\Release\\Plugins</value>
</setting>

<setting name="ForcedLocalIP" serializeAs="String">
    <value />
</setting>

NOTE: the ForcedLocalIP only needs to be used when there are multiple network adaptors on the PC and there isn't just a flat network. Basically, we have to figure out which adapter to get the PCs IP address from, to then pass it to the Pelco device so it knows where to send the stream to.

If things aren't streaming its best to put the local IP in here and retry.

  • Where required put the System Manager's details into VTAS.
    • VTAS menu -> Admin -> VTAS Pro -> Pelco Endura.

Pelco Analytics

As of VTAS 8.12 we now support Pelco Analytics. This section describes how to configure this in VTAS and doesn't cover the setup of the Pelco camera. Its recommended to seek out the documentation for the camera you are dealing with for that.

The configuration is in two parts. Firstly, we configure Pro Server to listen for the alarms, and then we move on to setting this up in VTAS.

Pro Server Configuration

In the Config file set the following settings:

  • ForceLocalIP,. needs to be the machines local IP (Eg. 192.168.1.1). It is passed to the cameras so they know where to send the analytic events.
  • PelcoAnalyticsPort, is the port the cameras are to send the analytic events to. This can be any unused port but 9017 has been set as the default.

VTAS

Pelco use a unique device number (UDN) to identity its cameras. We use the cameras IP and port. Therefore to link the two together in VTAS the camera record needs to hold the UDN of the camera. NOTE: This is only for cameras that need you need Analytic events from.

On the camera record enter the UDN in the Camera Device ID field prefixed with "uuid:" as shown below.

Obtaining the UDN can be quite tricky. You can use either the Pelco Utils tool "Endura Utilities" which will normally be readily available on any Pelco site from the engineer. It this tool isn't available then you can use 'Device Spy'. This can be found by searching the internet or here:

http://www.meshcommander.com/upnptools

Note that in all tools I've used to obtain the device number, there is a tree structure you must navigate. Rather than click on the top level, expand the top node and look for "Pelco Module Pelco", then expand this node and find the Camera device listed again. Its on this node you get the correct UDN. A reliable way to know you have the correct number is that it will end with a 4 as shown below:

uuid:0b89f811-a412-4d05-3ff3-0fdb8e676004

Following this, setting up the Pro Events is simple and much like any other Pro Event Setup. Select 'Pelco Analytics' from the trigger dropdown. Then select a camera and fill in any other Pro Event options necessary.

There is also a Camera Zone field. This relates to the zone that's configured on the camera for the analytics. -- When setting up Analytics on the camera is possible to configure more than one zone in a scene. Using this field in VTAS allows for a Pro Event to catch only the events triggered by a specific zone.
