Viewscape 11.37.1 Manual

SNMP

NOTE: Only v1 and v2 of SNMP is supported, v3 is not.

Prerequisites

A folder of 'base mibs' is needed, the default location for these is "i-Comply/VTAS/Base Mibs", this location can be changed via the desktop setting "SNMPBaseMibsPath".

The files that are required to be in this folder are kept in the Technical OneDrive under "Deployment/4 Integrations/SNMP".

Extra mib file(s) will likely be required for the specific device that will be firing the SNMP Traps, these can usually be found on google.

Configuration

The starting place for configuring SNMP is in VTAS under: >Admin -> Command & Control -> Third Party -> SNMP

This window is used to find the TRAPs that we want to listen for in the overall structure of mibs. Upon opening a mib file using the "Open Mib File" dialog, it will also open all of the mib files in the Base Mibs folder. This is because to listen for a specific trap we need its entire Object Identifier (OID), and the specific vendor mib files themselves only contain part of this.

The most likely place for your specific mib file to have its data is under: >iso -> org -> dod -> internet -> private -> enterprises

The only snmp entity type that is supported by the current integration is a TRAP type. These types are highlighted in green in the treeview.

Double clicking one of these highlighted nodes will create a device that will listen for that specific TRAP, it will open the device form with the event filled in with the correct OID, a pro event will need to be created and assigned to it though.

The connection details of the machine that will be sending the SNMP traps also need to be entered on this device, both IP address and Port.

Once this device is created, if a new SNMP port is defined, a pro server restart will be required for it start up a listener for this port.
